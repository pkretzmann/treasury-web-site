# 9.1 — Bogføring af aktieudbytte

Aktieudbytte bogføres via **Værdipapirafkastkladden**. Du kan udfylde kladden manuelt eller lade systemet foreslå linjer for alle relevante værdipapirer via batch-kørslen *Foreslå udbyttelinjer*.

---

## Manuelt — udfyldelse af kladden

### Åbn Værdipapirafkastkladden

Vælg **Security Return Journals** i Treasury-menuen i rollecentret. Hvis du har oprettet flere kladdetyper af typen *Afkast*, vises en oversigt. Vælg den ønskede kladdetype. Øverst i vinduet kan du skifte kladdenavn (batch) i feltet **Kladdenavn**.

### Udfyldelse af kladdelinjen

| Felt | Bemærkning |
|------|------------|
| **Handelsdato** | Datoen for udbyttebegivenheden. Bogføringsdatoen sættes automatisk lig med handelsdatoen. |
| **Bilagsnr.** | Hentes automatisk fra kladdenavnets nummerserie, hvis denne er konfigureret. Ellers angives manuelt. |
| **Bilagstype** | Sættes til **Afkast**. |
| **Værdipapirnummer** | Vælg aktien udbyttet vedrører. Når feltet udfyldes, indsættes automatisk beskrivelse, valutakode og *Markedsværdiprincip* (afledt af værdipapirets bogføringsgruppe). |
| **Afkast før skat** | Bruttoudbyttet i værdipapirets valuta. Feltet er obligatorisk. |
| **Udbytteskat** | Den indeholdte udbytteskat i værdipapirets valuta. Kan beregnes automatisk på baggrund af *Udbytteskat Pct.* fra bogføringsgruppen ved kørsel af batch-funktionen (se nedenfor). |
| **Beløb** | Beregnes automatisk som *Afkast før skat − Udbytteskat* (nettobeløbet). Ikke redigerbart. |
| **Beløb (RV)** | Nettobeløbet i regnskabsvaluta. Beregnes automatisk ud fra valutafaktoren. |
| **Depotbeholdningskode** | Vælg det depot, udbyttet vedrører. Modkontotype og modkonto sættes automatisk fra depotopsætningen. |
| **Modkontotype / Modkontonr.** | Udfyldes automatisk fra depotet (typisk en bankkonto). |

> **Bemærk:** Felterne *Antal*, *Fondskurs* og *Omkostninger* er ikke synlige på siden for afkastkladden, da de ikke er relevante for udbyttebogføring. *Markedsværdiprincip* afledes automatisk af værdipapirets bogføringsgruppe og er skrivebeskyttet.

---

## Automatisk forslag — *Foreslå udbyttelinjer*

For at oprette afkastlinjer for alle relevante aktier i porteføljen kan du køre rapporten **Suggest Dividend Lines** (`Dividend Batch`).

Vælg **Suggest Dividend Lines** i Treasury-rollecentrets handlinger under *Periodiske aktiviteter*. Anfordringsvinduet indeholder:

| Felt | Bemærkning |
|------|------------|
| **Posting Date** | Bogføringsdato for de dannede linjer. |
| **Document No.** | Bilagsnummer. Kan også hentes fra kladdens nummerserie. |
| **Yield Per Share** | Foreslået udbyttebeløb pr. aktie (FCY). |
| **Journal Template / Journal Batch** | Kladdetype og batch, hvor linjerne dannes. |
| **Filter på Værdipapir** | Begrænsning til ét eller flere værdipapirer. |

For hvert værdipapir oprettes én linje pr. depot i den valgte batch:
- *Antal* hentes fra depotets nettobeholdning.
- *Afkast før skat* beregnes som *Antal × Yield Per Share*.
- *Udbytteskat* beregnes som *Afkast før skat × Udbytteskat Pct.* fra bogføringsgruppen.

Brugeren reviewer linjerne og bogfører manuelt fra kladden.

## Bogføring

Klik på **Bogfør** (F9). Programmet kontrollerer inden bogføring:
- At Værdipapirnummer er udfyldt.
- At Bogføringsdato og Bilagsnr. er udfyldt.
- At Afkast før skat ikke er 0.
- At Valutafaktor er udfyldt, hvis der er angivet en fremmed valutakode.

### Resultatet af bogføringen

Ved et udbytte opretter programmet følgende poster:

| Posteringstype | Beskrivelse |
|----------------|-------------|
| **Værdipapirpost** | En post med bilagstype **Afkast**, antal = 0, afkast før skat, udbytteskat og nettobeløb. Beløbene gemmes med negativt fortegn på posten (svarer til kreditposteringen i finans). |
| **Detaljeret gevinstpost (Yield)** | Én linje pr. bruttoafkastandel der registrerer afkastet pr. detail (Profit Type = Yield). |
| **Detaljeret gevinstpost (Dividend Tax)** | Linje for udbytteskat (Profit Type = Dividend Tax), hvis Udbytteskat ≠ 0. |
| **Finanspost (modkonto)** | Modkontoen (typisk bank) debiteres med nettobeløbet i regnskabsvaluta. |
| **Finanspost (afkastkonto)** | Afkastkontoen (fra Værdipapir gevinstbogføringsgruppe for *Realization = Realized*, *Document = Yield*, *Profit Type = Yield*) krediteres med bruttoafkastet i regnskabsvaluta. |
| **Finanspost (udbytteskatkonto)** | Udbytteskatkontoen (fra Værdipapir gevinstbogføringsgruppe med *Profit Type = Dividend Tax*) debiteres med udbytteskatten i regnskabsvaluta — kun hvis Udbytteskat ≠ 0. |

> **Bemærk:** *Realiseringstype* (Realiseret / Urealiseret) udfyldes manuelt på kladdelinjen. Ved Urealiseret afkast bogføres mod en separat profitkonto (Realization = Unrealized), og afkastet tilbageføres typisk i samme kladde eller ved næste periodisering — se 9.3 for periodiseret obligationsrente.
