Fanebladet Generelt

Når du opretter en ny obligation, starter du på fanebladet Generelt. Her angiver du obligationens grundoplysninger og lader systemet automatisk generere obligationens navn baseret på de værdier, du udfylder.

Felt Udfyldes Bemærkning
No. V/T Tildeles automatisk fra nummerserien, hvis denne er konfigureret. Er feltet tomt, angives nummeret manuelt.
Security Type T Vælg Bond (Obligation).
ISIN Code V Angiv eller vælg ISIN-koden. Oprettes automatisk som ny ISIN-kode, hvis koden ikke kendes.
Issuer Code V Vælg den relevante udsteder. Feltet er fremhævet som obligatorisk, når Security Type = Bond. Benyttes i automatisk navnformatering. Se afsnittet om automatisk generering af navn nedenfor.
Name V Angiv obligationens navn eller lad systemet generere det automatisk. Se afsnittet om automatisk generering af navn nedenfor.
Search Name V Udfyldes automatisk fra Name, men kan redigeres manuelt. Vises kun via Vis mere. Feltet må ikke stå blankt.
Currency Code V Angiv valutakode, hvis obligationen ikke er noteret i regnskabsvalutaen. Lad feltet stå blankt ved DKK.
Remaining Quantity P Beregnes og opdateres automatisk på baggrund af bogførte poster.
Remaining Value P Beregnes og opdateres automatisk på baggrund af bogførte poster.
Blocked V Markér for at spærre obligationen. En spærret obligation kan hverken vælges på en kladdelinje eller bogføres.
Forkortelser i kolonnen "Udfyldes": T = Tvungen (skal altid udfyldes), V = Valgfri (udfyldes efter behov), P = Programstyret (vedligeholdes automatisk, kan ikke redigeres).

Automatisk generering af navn

Systemet kan automatisk generere obligationens navn ud fra en skabelon. Hvis du har konfigureret formatering af obligationsnavn (se Opsætning af værdipapirer), vil systemet automatisk formatere navnet første gang, når du udfylder de relevante felter.

Skabelonen bruger følgende pladsholdere:

  • %1 = Din egen tekst (hvad du selv skriver i Name-feltet)
  • %2 = Udsteder (fra Issuer Code)
  • %3 = Nominel renteprocent (fra Nom. Interest Rate %)
  • %4 = Udløbsår (fra L. Coup. Payment Date)

Eksempel: Hvis skabelonen er "%1 %2 %3% %4" og du udfylder "Obl", udsteder "Danske Bank", rentesats "3,5%" og sidste kupondag "2030", genereres navnet automatisk til "Obl Danske Bank 3,5% 2030".

Vigtigt — manuel ændring af navn: Når du først har oprettet obligationen med automatisk navn, vil senere manuelle ændringer af Name-feltet ikke blive overskrevet af formateringen. Hvis du vil regenerere navnet på ny, skal du rydde Name-feltet helt og udfylde det igen.

Fanebladet Bond Details

Fanebladet Bond Details vises kun, når du har valgt Bond som Security Type. Her angiver du obligationens specifikke egenskaber — rente, kuponer og særlige karakteristika.

Felt Udfyldes Bemærkning
Nom. Interest Rate % V Angiv den nominelle renteprocent. Bruges i renteberegning. Denne værdi kan også bruges i automatisk generering af navn (se %3 ovenfor).
Payment Code V Angiv terminskoden for obligationens rentebetalingsplan. Denne kode definerer, hvor ofte renter betales (f.eks. årligt, halvårligt osv.).
F. Coup. Payment Date V Angiv første kuponforfaldsdag — dvs. den første dato, hvor der udbetales renter på obligationen.
L. Coup. Payment Date V Angiv sidste kuponforfaldsdag (udtræksdato). Dette er obligationens slutdato, hvor den sidste rentebetaling og tilbagebetaling af hovedstolen sker. Denne værdi bruges også i automatisk generering af navn (se %4 ovenfor).
Index Bond V Markér, hvis det er en indeksobligation. Indeksobligationer er obligationer, hvor renter eller hovedstol er knyttet til et indeks (f.eks. inflationsindeks).
Excl. Interest Calculation V Markér, hvis obligationen skal udelades ved kørsel af renteberegningsbatchen. Brug dette, hvis obligationen f.eks. er udløbet eller skal håndteres manuelt.

Fanebladet Bogføring

På fanebladet Bogføring knytter du obligationen til finanskontoplanen og definerer bogføringsgrupper og dimensioner. Disse oplysninger styrer, hvordan posteringer på obligationen påvirker bogholderiet.

Felt Udfyldes Bemærkning
Posting Group T Angiv bogføringsgruppen, der knytter obligationen til finanskontoplanen. Se vejledningen om bogføringsgrupper.
Global Dimension 1 Code T Angiv den primære dimensionsværdi. Obligatorisk, hvis dimensioner er konfigureret.
Global Dimension 2 Code T Angiv den sekundære dimensionsværdi.
Report Grouping Method V Grupperingsmetode til rapporter: Posting Group, Dimension Value eller Detailed information.
Report Grouping Code V Tilhørende gruppekode — afhænger af valgt metode.

Praktisk vejledning til oprettelse

Trin-for-trin guide

  1. Angiv Security Type som Bond — fanebladet Bond Details bliver nu synligt.
  2. Udfyld Issuer Code — dette er obligatorisk for obligationer og bruges også til automatisk generering af navn.
  3. Angiv obligationens grundlæggende egenskaber: Nom. Interest Rate %, F. Coup. Payment Date og L. Coup. Payment Date.
  4. Lad Name-feltet være og lade systemet automatisk generere navnet baseret på skabelonen — eller angiv selv navn, hvis ingen skabelon er konfigureret.
  5. Tilknyt obligationen til finanskontoplanen via fanebladet Bogføring — udfyld Posting Group, Global Dimension 1 Code og Global Dimension 2 Code.
  6. Gem obligationen.

Hvad er obligatorisk?

  • Security Type: Skal være Bond.
  • Posting Group: Knyt obligationen til en bogføringsgruppe.
  • Global Dimension 1 Code og Global Dimension 2 Code: Fyld ud, hvis dimensioner er konfigureret.
  • Issuer Code: Obligatorisk for obligationer.

Hvad kan du udfylde efter behov?

  • Kuponer: Udfyld F. Coup. Payment Date og L. Coup. Payment Date, hvis du vil bruge systemets rentebetaling.
  • Payment Code: Definer betalingsplanen (årligt, halvårligt osv.).
  • Index Bond: Markér, hvis obligationen er indekseret.
  • Excl. Interest Calculation: Markér, hvis obligationen skal udelades fra renteberegning.
Vigtigt — ændring af navn: Hvis du ændrer Name-feltet manuelt efter oprettelse, vil systemet ikke automatisk regenerere det ved næste gemning. Ryd Name-feltet helt og udfyld det igen, hvis du vil have systemet til at anvende navnskabelonen på ny.
